Choose your upgrade method

  • The current GaianDB installation is on a test environment and can be replaced without impacting services:

    • Follow Upgrade the current GaianDB installation.

  • The current GaianDB installation is on a production environment and cannot be replaced without impacting services:

    • Follow Migrate by installation GaianDB alongside the current installation.

Upgrade the current GaianDB installation

Upgrade GaianDB without keeping the previous version.
This installation doesn't allow for a demigration scenario. If in doubt use the Migrate by installation GaianDB alongside the current installation. method.

  1. Get the new GaianDB installation zip file, copy it and unzip it on the machine you wish to upgrade.
  2. Stop the GaianDB nodes running on the machine.
  3. Copy all the files in the lib directory into your installation's lib directory.
  4. Review the GaianDB documentation and copy any new scripts provided into your current installation.
  5. Restart the nodes.
  6. Review the rest of the documentation to read about the new functionality and how to take advantage of it.

Migrate by installation GaianDB alongside the current installation

Upgrade alongside the current GaianDB installation

Install GaianDB in parallel to the current installation, for review and testing, before deployment.
This installation allows to keep both current and new environment allowing for a demigration if necessary.

  1. Get the new GaianDB installation zip file, copy it on the machine you wish to upgrade.
  2. Unzip the file in a location close to the current installation, rename the directory if needed, for example GaianDB-new.
  3. Copy the following files from the current GaianDB installation into the new directory:
    • All the gaiandb_config.properties files used by the nodes running from this installation.
    • All the third-party libraries added to the lib directory.
    • Any other third-party libraries added to the installation if not all in the lib directory.
    • Any files (Excel, CSV..) added to the installation.
    • Any scripts added to the installation.
    • The GaianDB startup scripts, if they have been modified to load third-party libraries.
  4. Review the GaianDB documentation to read about the new functionality and how to take advantage of it.
  5. Test the setup as needed.

When you are reading to migrate, you can backup the previous GaianDB installation and move over to the new one, see below.

Move to the new installation when ready

  1. Stop the GaianDB nodes running on the machine.
  2. Backup your current installation, for example by renaming the GaianDB installation directory to GAIANDB-backup.
  3. Either:
    • Unzip the new GaianDB zip file, and copy it to replace the directory you just renamed.
      • Or
    • Use the new installation directory if you following the instructions above to install GaianDB in parallel.
  4. Copy the following files from the GAIANDB-backup into the new GAIANDB installation:
    • All the gaiandb_config.properties files used by the nodes running from this installation.
    • All the third-party libraries added to the lib directory.
    • Any other third-party libraries added to the installation if not all in the lib directory.
    • Any files (Excel, CSV..) added to the installation.
    • Any scripts added to the installation.
    • The GaianDB startup scripts, if they have been modified to load third-party libraries.
  5. Restart the nodes.
  6. Review the GaianDB documentation to read about the new functionality and how to take advantage of it.
